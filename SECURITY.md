# Security Policy

## Supported Versions

We actively maintain and provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

Security updates are provided for the latest minor version within the current major version. We recommend keeping your installations up to date.

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

### How to Report

1. **Email us directly** at: **security@truthchat.in**

2. **Include the following information**:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact and severity assessment
   - Suggested fix or mitigation (if available)
   - Your contact information (optional, for follow-up questions)

3. **What to expect**:
   - We will acknowledge receipt of your report within **48 hours**
   - We will provide an initial assessment within **7 business days**
   - We will keep you informed of our progress and resolution timeline
   - We will notify you when the vulnerability has been addressed

### What NOT to Include

When reporting security vulnerabilities, please **do not**:
- Include sensitive data, credentials, or personally identifiable information
- Share the vulnerability publicly before we've had a chance to address it
- Use the vulnerability to access systems or data without authorization
- Disclose details in public GitHub issues, discussions, or pull requests

---

## Responsible Disclosure

We follow responsible disclosure practices:

1. **Initial Report**: Report the vulnerability privately via email
2. **Verification**: We verify and assess the vulnerability
3. **Resolution**: We develop and test a fix
4. **Disclosure**: We coordinate public disclosure after a fix is available
5. **Credit**: We acknowledge responsible reporters (with permission)

### Safe Harbor

Security research activities conducted in good faith, including:
- Testing our publicly available services and repositories
- Reporting vulnerabilities responsibly
- Following this security policy

...are considered authorized conduct. We will not pursue legal action against security researchers who comply with this policy.

---

## Security Best Practices

When using our software or services:

- **Keep installations updated** to the latest supported version
- **Use strong authentication** where available
- **Follow principle of least privilege** in access controls
- **Monitor for security advisories** and apply patches promptly
- **Report suspicious activity** to security@truthchat.in

---

## Scope

This security policy applies to:
- All public repositories under the TruthChat organization
- Public-facing services and APIs
- Documentation and public resources

This policy does **not** cover:
- Social engineering attacks
- Physical security issues
- Denial of service attacks
- Issues in third-party dependencies (please report to the respective maintainers)

---

## Security Updates

Security updates are released as:
- **Patch releases** for critical vulnerabilities
- **Minor releases** for important security fixes
- **Security advisories** published in repository releases and documentation

We recommend subscribing to repository notifications to receive security update announcements.
